# Исследование проекта (Research)

## Анализ предметной области
Академия Развития Человека проводит оффлайн и онлайн мероприятия (семинары, вебинары, расстановки, курсы). Специфика бизнеса требует гибкого управления ресурсами (спикеры, залы) и глубокой аналитики по участникам.

## Технологический стек
- **Frontend:** React + Tailwind CSS (для основного приложения), Чистый HTML/CSS/JS (для лендинга).
- **Backend/DB:** Firebase (Firestore для данных, Auth для верификации).
- **Голосовой ввод:** Web Speech API.
- **Анимации:** Motion.

## Анализ рисков
1. **Конфликты расписания:** Сложная логика пересечений требует надежной проверки на стороне сервера/клиента.
2. **Мобильный UX:** Администраторы работают "на бегу", интерфейс должен быть максимально упрощен.
3. **Безопасность данных:** Финансовая информация и PII участников требуют строгих правил Firestore.

---

## Экспертная оценка

### Эксперт 1: Специалист по UX/UI (Mobile First)
*Рекомендации:*
- Использовать "карточный" дизайн для расписания в мобильной версии.
- Кнопки записи должны быть в зоне досягаемости большого пальца.
- Голосовой ввод должен сопровождаться визуальным подтверждением распознанного текста перед сохранением.
- Цветовую гамму логотипа использовать как акценты, чтобы не перегружать интерфейс "радугой".

### Эксперт 2: Системный архитектор (Backend & Data)
*Рекомендации:*
- Реализовать денормализацию данных: хранить имя спикера и название локации прямо в объекте события для быстрой фильтрации.
- Реферальную программу строить на уникальных ID, записываемых при регистрации.
- Для проверки конфликтов создать отдельный сервис-валидатор, который проверяет пересечение временных интервалов.

### Эксперт 3: Бизнес-аналитик (Analytics & Finance)
*Рекомендации:*
- Статистику выводить в виде дашбордов с графиками (Recharts/D3).
- Ввести понятие "Филиал" как обязательный атрибут участника и мероприятия для сегментации доходов.
- Автоматизировать расчет бонусов по реферальной программе для прозрачности выплат.

---

## Общий список рекомендаций
1. Реализовать систему уведомлений о конфликтах в реальном времени при создании события.
2. Обеспечить бесшовный переход между лендингом и личным кабинетом.
3. Использовать PWA-подход для удобства работы администраторов без установки приложения из сторов.
4. Внедрить систему статусов участников (Новичок, Активный, Мастер и т.д.) на основе посещаемости.
