# Руководство по тестированию (TESTING)

## 1. Подготовка
1. Убедитесь, что у вас есть доступ к интернету.
2. Откройте приложение по предоставленной ссылке.
3. Для тестирования админ-панели используйте тестовый аккаунт (будет предоставлен после настройки Firebase).

## 2. Тест-кейсы

### 2.1. Публичный лендинг (`index.html`)
- [ ] Откройте `index.html` в браузере.
- [ ] Проверьте адаптивность: сузьте окно браузера до мобильного размера. Меню и блоки должны перестроиться.
- [ ] Проверьте наличие всех блоков из ТЗ (Расписание, О нас, Контакты).
- [ ] Убедитесь, что нет внешних запросов к JS-библиотекам (проверка через Network tab).

### 2.2. Расписание (React App)
- [ ] Перейдите в раздел расписания.
- [ ] Попробуйте отфильтровать мероприятия по месяцам.
- [ ] Проверьте, скрываются ли прошедшие мероприятия по умолчанию.
- [ ] Нажмите "Записаться" (требуется авторизация).

### 2.3. Админ-панель и Конфликты
- [ ] Попробуйте создать два мероприятия для одного спикера на одно и то же время в разных местах. Должна появиться ошибка.
- [ ] Попробуйте занять один зал двумя мероприятиями одновременно. Должна появиться ошибка.
- [ ] Проверьте голосовой ввод: нажмите на иконку микрофона в форме создания события и произнесите параметры (дата, название).

### 2.4. Реферальная программа
- [ ] Скопируйте реферальную ссылку из профиля.
- [ ] Зарегистрируйте нового пользователя по этой ссылке.
- [ ] Проверьте, начислился ли бонус пригласившему.

### 2.5. Финансы и Статистика
- [ ] В админке введите данные об оплате участника.
- [ ] Проверьте, обновились ли графики в разделе статистики.
- [ ] Сравните данные с предыдущим месяцем (используйте тестовые данные).

## 3. Исправление ошибок
Если вы нашли баг:
1. Опишите шаги воспроизведения.
2. Приложите скриншот консоли (F12).
3. Передайте разработчику для внесения правок в код.
