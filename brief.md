# Техническое задание (Brief)

## 1. Идея проекта
Единая цифровая экосистема для Академии Развития Человека, объединяющая публичный лендинг для привлечения клиентов и внутреннюю систему управления (ERP/CRM) для администраторов, спикеров и участников.

## 2. Данные исследования
Исследование показало необходимость Mobile-First подхода. Основные пользователи (администраторы) работают в мобильном интерфейсе. Важна автоматизация контроля ресурсов (спикеры/залы) для исключения человеческого фактора при планировании.

## 3. Функциональные требования
### 3.1. Публичный лендинг (index.html)
- Презентация академии.
- Актуальное расписание на текущий месяц.
- Форма быстрой связи.
- Кнопка перехода в личный кабинет/регистрацию.
- Дизайн: Чистый HTML/CSS, адаптивный, цвета логотипа.

### 3.2. Личный кабинет участника
- Профиль с историей посещений.
- Информация о наставнике и приглашенных (рефералах).
- Текущий бонусный баланс.
- Запись на новые мероприятия с верификацией.

### 3.3. Панель администратора
- Управление расписанием (добавление, редактирование, удаление).
- **Голосовой ввод** для быстрого создания событий.
- Валидация конфликтов:
  - Спикер в двух местах одновременно -> Ошибка.
  - Зал занят двумя событиями -> Ошибка.
  - Параллельные события в разных местах -> Предупреждение.
- Учет финансов: Оплаты (доход) и Расходы (аренда, гонорары).
- Дашборды со статистикой (сравнение периодов).

### 3.4. Реферальная программа
- Генерация реферальных ссылок/кодов.
- Автоматическое начисление бонусов при регистрации и оплате нового участника.

## 4. Дизайн-система
- **Основной цвет:** Белый/Светло-серый (фон).
- **Акцентные цвета:** Фиолетовый (#9333ea), Синий (#2563eb), Зеленый (#16a34a), Красный (#dc2626).
- **Шрифты:** Sans-serif (Inter/Roboto) для читаемости.

## 5. Технические ограничения
- Использование Firebase для БД и Auth.
- Отсутствие внешних библиотек в `index.html`.
- Полная адаптивность под Mini-app (Telegram WebApp и аналоги).
